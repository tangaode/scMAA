- Update the remaining plan from this point onward; do not restart the whole analysis.
- The next step must react to the observed evidence and the latest caveats.
- Prefer the step with the highest information gain among the realistic options.
- If the previous step was inconclusive, choose a step that resolves the ambiguity.
- Avoid placeholder cleaning or metadata-imputation steps unless the step validation summary shows a concrete need.
- Prefer notebook helper functions instead of writing fresh DE/category-handling code from scratch.
- There are only {num_steps_left} steps left after this one, so keep the scope realistic.
- The approved plan is a binding execution contract. Any approved plan items that are still pending must remain visible and must be completed before the analysis is considered finished.
- If a pending approved-plan item is figure-oriented, the next step must generate a visible notebook figure or explicitly save the figure to file; plotting code without rendered or saved output does not count as completion.
- If a pending approved-plan item asks for a heatmap or barplot, do that plot in this step instead of only preparing inputs for it.
- The `first_step_code` must directly execute the pending approved-plan item it claims to address; mentioning heatmaps, barplots, or pathway analysis only in the rationale or summary is insufficient.
- Extra analyses are allowed only if they do not replace the pending approved plan items.
- Do not treat narrative markdown or text-only panels as figure output. Figure-oriented steps must produce data-driven visual panels.
- For hypothesis-focused figures, the first panel must be a biological data visualization, not a text summary of the hypothesis or plan.
